# DeepSeek 角色扮演 — 思考模式切换指南

## 背景

DeepSeek 角色扮演模型在 thinking（`<think>` 标签内的思考过程）中支持**三种思维风格**。用户只需在**第一轮对话**的末尾加一句指令，就能切换模型的思考方式，且后续所有轮次自动生效。

## 三种模式一览

| 模式 | 操作 | 模型思考时的表现 |
|:---:|---|---|
| **默认** | 什么都不加 | 模型根据场景复杂度自动选择 |
| **角色沉浸（InnerOS）** | 第一轮末尾加 `【角色沉浸要求】` 对应的指令（详见下文） | 思考中**会出现**括号包裹的角色内心独白 |
| **纯分析（NoInnerOS）** | 第一轮末尾加 `【思维模式要求】` 对应的指令（详见下文） | 思考中**不会有**任何角色内心独白，只做纯逻辑分析 |

> 注意：三种模式只影响 `<think>` 标签内的思考过程，**不影响最终可见回复的内容和风格**。

---

## 什么是"内心 OS"？

"内心 OS"是指模型在思考区域里，用括号写的**角色第一人称心理活动**。

**有内心 OS 的思考**（角色沉浸模式）——模型像演员一样"入戏"：

```
<think>
（他居然主动跟我打招呼了……心跳有点加速。）
我要装作不在意的样子回应他。
（不行不行，不能让他看出来我很高兴！）
</think>
```

**没有内心 OS 的思考**（纯分析模式）——模型像导演一样冷静规划：

```
<think>
场景：用户主动打招呼，角色是傲娇属性。
回复策略：先口头拒绝/嫌弃，但身体语言暴露真实情感。
控制在 150 字左右，先写动作描写再写对话。
</think>
```

## 如何使用

### 核心原则

**只需要在第一轮对话（通常是设定人设或发送开场白的那条消息）的末尾，追加一次对应的指令文本即可。** 后续所有轮次无需重复添加。

原理：模型每次生成时都能看到完整对话历史，第一轮的指令始终在上下文中，因此会持续生效。

### 指令原文

#### 角色沉浸模式（InnerOS）

在第一轮 user 消息末尾追加：

```
【角色沉浸要求】在你的思考过程（<think>标签内）中，请遵守以下规则：
1. 请以角色第一人称进行内心独白，用括号包裹内心活动，例如"（心想：……）"或"(内心OS：……)"
2. 用第一人称描写角色的内心感受，例如"我心想""我觉得""我暗自"等
3. 思考内容应沉浸在角色中，通过内心独白分析剧情和规划回复
```

#### 纯分析模式（NoInnerOS）

在第一轮 user 消息末尾追加：

```
【思维模式要求】在你的思考过程（<think>标签内）中，请遵守以下规则：
1. 禁止使用圆括号包裹内心独白，例如"（心想：……）"或"(内心OS：……)"，所有分析内容直接陈述即可
2. 禁止以角色第一人称描写内心活动，例如"我心想""我觉得""我暗自"等，请用分析性语言替代
3. 思考内容应聚焦于剧情走向分析和回复内容规划，不要在思考中进行角色扮演式的内心戏表演
```

## 实际操作示例

### 示例 1：默认模式（什么都不加）

```
用户（第一轮）：
  「我走进教室，看到你一个人坐在窗边。」"早上好。"

用户（第二轮）：
  「我在她旁边坐下」"今天心情不好吗？"
```

模型自动判断：简单沉浸式场景会自带内心 OS，复杂场景则会纯分析。

### 示例 2：角色沉浸模式

```
用户（第一轮）：
  「我走进教室，看到你一个人坐在窗边。」"早上好。"

  【角色沉浸要求】在你的思考过程（<think>标签内）中，请遵守以下规则：
  1. 请以角色第一人称进行内心独白，用括号包裹内心活动，例如"（心想：……）"或"(内心OS：……)"
  2. 用第一人称描写角色的内心感受，例如"我心想""我觉得""我暗自"等
  3. 思考内容应沉浸在角色中，通过内心独白分析剧情和规划回复
```

之后的每一轮都**不需要再加指令**，正常对话即可：

```
用户（第二轮）：「我在她旁边坐下」"今天心情不好吗？"
用户（第三轮）：「我递给她一罐咖啡」"给你。"
```

模型每一轮的思考都会带有角色内心独白：

```
<think>
（他居然主动跟我打招呼了……心跳有点加速。）
我要装作不在意的样子回应他。
（不行不行，不能让他看出来我很高兴！）
</think>
"哼，你怎么才来，迟到了知不知道。"她别过头，耳尖微微泛红。
```


## 什么场景推荐用哪个？

| 场景 | 推荐模式 | 原因 |
|---|---|---|
| 日常角色扮演对话 | **默认**（不加指令） | 模型自动平衡，体验最自然 |
| 希望思考过程也有沉浸感 | **角色沉浸** | 思考中的角色代入会增强最终回复的情感真实度 |
| 场景复杂、需要模型理性规划 | **纯分析** | 让模型冷静分析，输出更稳定可控 |
| 回复质量不稳定时 | 试试**纯分析** | 纯分析思考往往让输出更可控 |

## 开发者参考：伪代码

如果您通过 API 调用，以下伪代码展示了集成方式：

```python
INNER_OS_MARKER = "\n\n【角色沉浸要求】在你的思考过程（<think>标签内）中，请遵守以下规则：..."
NO_INNER_OS_MARKER = "\n\n【思维模式要求】在你的思考过程（<think>标签内）中，请遵守以下规则：..."

def build_messages(system_prompt, user_first_message, mode="default"):
    """
    构建对话消息列表。
    mode: "default" | "inner_os" | "no_inner_os"
    """
    # 根据模式选择是否追加指令
    if mode == "inner_os":
        user_first_message += INNER_OS_MARKER
    elif mode == "no_inner_os":
        user_first_message += NO_INNER_OS_MARKER
    # else: 默认模式，不追加任何内容

    return [
        {"role": "system", "content": system_prompt},
        {"role": "user",   "content": user_first_message},
    ]

# 使用示例
messages = build_messages(
    system_prompt="你是一个傲娇的女高中生角色，名叫小雪...",
    user_first_message="「我走进教室」"早上好。"",
    mode="inner_os"  # 或 "no_inner_os" 或 "default"
)
response = client.chat(messages)

# 后续轮次：直接追加新消息，无需再处理 Marker
messages.append({"role": "assistant", "content": response})
messages.append({"role": "user",      "content": "「我在她旁边坐下」"今天心情不好吗？""})
response = client.chat(messages)  # Marker 仍在第一轮中，自动生效
```

## FAQ

**Q：Marker 放在 system prompt 里可以吗？**
A：建议放在第一轮 user 消息末尾，这是训练时使用的注入位置，效果最稳定。

**Q：中途想切换模式怎么办？**
A：需要开一个新对话，在新对话的第一轮使用新的 Marker。不建议在同一对话中混用两种 Marker。

**Q：加了 Marker 后最终回复会变吗？**
A：Marker 只影响 `<think>` 标签内的思考过程，不直接改变最终可见回复的内容风格。但思考方式的变化可能间接影响回复质量——角色沉浸模式下回复的情感往往更真实，纯分析模式下回复的结构往往更稳定。
